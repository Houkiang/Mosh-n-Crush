# 联机改造清单

> 项目：`houkeliang`（类吸血鬼幸存者 3D）  
> 目标：在尽量保留现有架构的前提下，完成 2-4 人 PvE 联机（建议 Host-Authoritative）

## 0. 先决策（未确认前不要进入大规模开发）

- [ ] 选择网络方案：`Netcode for GameObjects (NGO)` / `Mirror` / 其他
- [ ] 选择会话模型：`房主制 Host-Authoritative`（推荐）/ 独立服务器
- [ ] 房间人数上限：`2` / `4` / `更多`
- [ ] 平台目标：`仅 PC` / `PC + 移动`
- [ ] 是否需要断线重连：`MVP 不做` / `正式版做`

## 1. 架构改造（P0）

### 1.1 网络基础层
- [ ] 引入网络框架（安装包、基础 NetworkManager、传输层）
- [ ] 建立 `Bootstrap` 场景或初始化流程（主菜单 -> 房间 -> 游戏）
- [ ] 增加玩家连接/断开事件处理（玩家实体创建、销毁）

### 1.2 权威边界（必须明确）
- [ ] 规定“仅 Host 计算”的系统：刷怪、掉落、伤害、升级随机、波次推进
- [ ] 规定“客户端提交输入”的系统：移动输入、升级选择、UI操作请求
- [ ] 规定“仅本地显示”的系统：摄像机、HUD、手柄输入组件

### 1.3 单例与全局状态解耦
- [ ] 降低对单机单例强依赖（`GameManager.Instance`、`EnemySpawner.Instance` 等）
- [ ] 逐步把 `FindObjectOfType/FindGameObjectWithTag` 改为网络生命周期下的显式绑定
- [ ] 区分 `LocalPlayer` 与 `RemotePlayer` 引用来源

## 2. 玩家系统联机化（P0）

### 2.1 输入与移动
- [ ] `PlayerController`：仅本地拥有者读取输入（非拥有者禁用输入）
- [ ] 玩家位置/朝向同步（NetworkTransform 或自定义插值同步）
- [ ] 校正与插值策略（减少抖动、穿插）

### 2.2 玩家属性同步
- [ ] `Player` 核心状态网络化：HP、等级、经验、死亡状态、关键属性
- [ ] 保证血量扣减与经验增加由 Host 最终裁决
- [ ] 本地 HUD 仅订阅本地玩家状态，避免读到他人数据

### 2.3 相机与UI归属
- [ ] `CameraFollow` 绑定本地玩家
- [ ] `JoystickController` 仅在本地玩家生效
- [ ] `GameHUD`、`PlayerHealthBar` 做本地与远端显示分离

## 3. 战斗系统联机化（P0）

### 3.1 敌人与刷怪
- [ ] `EnemySpawner` 改为 Host 驱动（波次、计时、权重随机）
- [ ] 敌人生成由 Host 广播（客户端只接收结果）
- [ ] 敌人目标选择规则支持多玩家（最近/仇恨/随机）

### 3.2 敌人行为
- [ ] `EnemyMovement` 在 Host 执行 AI 与物理，客户端做位置插值展示
- [ ] `EnemyContactDamage`、`EnemyShooter`、`EnemyProjectile` 改为 Host 结算
- [ ] 统一敌人死亡事件来源，避免重复结算

### 3.3 武器与投射物
- [ ] `WeaponManager`、`WeaponBase` 改为 Host 触发攻击逻辑
- [ ] 子弹/近战命中改为 Host 判定，客户端仅播放表现
- [ ] `ShieldWeapon/ShieldProjectile` 明确碰撞权威（建议 Host）

### 3.4 伤害、击退与数字表现
- [ ] `IDamageable` 调用链统一走服务端裁决路径
- [ ] 伤害数字与受击表现改为事件广播（避免每端各算一遍）
- [ ] 处理高频事件的带宽压力（聚合/节流）

## 4. 掉落、升级、波次、结算（P1）

### 4.1 掉落与经验球
- [ ] `DropManager` 仅 Host 生成掉落
- [ ] `ExperienceOrb/MagnetArea` 由 Host 判定拾取与归属
- [ ] 多玩家拾取冲突规则（先到先得/共享经验）

### 4.2 升级系统
- [ ] `UpgradeManager`：升级候选随机改为 Host 生成并下发
- [ ] 升级选择通过 RPC/Command 提交，Host 应用
- [ ] 移除/替代全局 `Time.timeScale` 暂停方案（改为逻辑暂停或投票暂停）

### 4.3 波次与Boss UI
- [ ] `WaveInfoUI` 数据来源改为网络同步状态
- [ ] Boss 血条绑定网络实体，不直接依赖本地对象生命周期

### 4.4 游戏结束
- [ ] `GameOverUI` 改为会话状态驱动，不由本地 `OnPlayerDied` 直接全局停时
- [ ] 失败条件定义：全员死亡 / 房主死亡 / 倒地机制

## 5. 对象池与性能（P1）

- [ ] `PoolManager` 区分“网络对象生成”与“本地视觉对象池”
- [ ] 高频对象（子弹、伤害字）按可见性/距离做简化策略
- [ ] 压测目标：100~300 敌人 + 多玩家稳定帧率
- [ ] 网络指标监控：带宽、RTT、丢包下表现

## 6. 必修 Bug 与一致性修复（建议先做）

- [ ] 修复分数重复结算风险：`Enemy.Die()` 与 `GameManager.HandleScore()` 不应双加分
- [ ] 清理 `Time.timeScale` 相关的单机全局暂停路径
- [ ] 统一随机源（Host 唯一随机，或可复现随机种子）
- [ ] 明确死亡/回收顺序，避免对象池回收时序导致的状态错乱

## 7. 开发里程碑

### 里程碑 A（1-2 周）
- [ ] 玩家可联机进房
- [ ] 本地/远端玩家移动同步
- [ ] 基础 HUD 和相机归属正确

### 里程碑 B（2-4 周）
- [ ] Host 刷怪 + 敌人同步
- [ ] 玩家攻击/伤害/死亡闭环
- [ ] 基础波次推进可用

### 里程碑 C（4-6 周）
- [ ] 升级选择联机化
- [ ] 掉落与经验球一致性
- [ ] 基础结算流程

### 里程碑 D（6 周+）
- [ ] 断线重连（可选）
- [ ] 匹配/大厅（可选）
- [ ] 性能优化与反作弊基础

## 8. 验收清单（最小可玩联机）

- [ ] 2 名玩家可稳定联机 30 分钟不崩溃
- [ ] 敌人数量、位置、血量在两端一致
- [ ] 升级结果在两端一致
- [ ] 掉落与经验不会重复结算
- [ ] 游戏失败条件一致且可复现

## 9. 待你确认的问题（影响实现路径）

- [ ] 你更倾向 `NGO` 还是 `Mirror`？
- [ ] 经验分配要“个人拾取”还是“全队共享”？
- [ ] 升级时是否允许“仅升级玩家自己暂停”，还是“全队共同等待”？
- [ ] 敌人主要仇恨规则：最近玩家 / 伤害最高玩家 / 混合？
- [ ] MVP 是否接受“房主退出即房间结束”？
