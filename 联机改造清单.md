### B1 验收日志标准（直观版）

建议筛选命令：
- `findstr /i "B1 Launch StartServer StartClient result Event OnServerStarted OnClientConnectedCallback OnClientDisconnectCallback OnTransportFailure exception error" server.log`
- `findstr /i "B1 Launch StartServer StartClient result Event OnServerStarted OnClientConnectedCallback OnClientDisconnectCallback OnTransportFailure exception error" client1.log`

服务端通过信号（至少命中以下 3 条）：
- `[B1] Launch args parsed => mode=DedicatedServer, ip=..., port=...`
- `[B1] StartServer result => True, status=Server running @ ...`
- `[B1] Event => OnClientConnectedCallback, role=Server, clientId=...`（有客户端接入时）

客户端通过信号（至少命中以下 3 条）：
- `[B1] Launch args parsed => mode=Client, ip=..., port=...`
- `[B1] StartClient result => True, status=Client connecting -> ...`
- `[B1] Event => OnClientConnectedCallback, role=Client, clientId=...`（连接建立后）

断开重连通过信号：
- 关闭客户端后，服务端/客户端日志出现 `OnClientDisconnectCallback`
- 重新启动客户端后，再次出现 `OnClientConnectedCallback`

失败信号（命中任一条需回查）：
- `[B1] Event => OnTransportFailure`
- `StartServer result => False` 或 `StartClient result => False`
- `Port 7777 is likely already in use`

### B2 验收日志标准（网络玩家最小原型）

建议筛选命令：
- `findstr /i "B2 NetworkPlayer spawned NetworkPlayer despawned owner isOwner isServer isClient exception error" server.log`
- `findstr /i "B2 NetworkPlayer spawned NetworkPlayer despawned owner isOwner isServer isClient exception error" client1.log`

通过信号（核心）：
- 服务端日志出现多条：`[B2] NetworkPlayer spawned => ... owner=... isServer=True`
- 客户端日志出现一条本地拥有者：`[B2] NetworkPlayer spawned => ... isOwner=True isClient=True`
- 当第二个客户端加入时，已有客户端也能看到新增 `spawned` 日志（远端对象）
- 关闭客户端后出现：`[B2] NetworkPlayer despawned => ... owner=...`

失败信号：
- 出现 `has no NetworkObject assigned`
- 出现 `Deferred messages were received ... OnAddPrefab` 超时
- 出现 `Spawn` 相关 `error/exception`

### B3 验收日志标准（Owner 输入所有权）

建议筛选命令：
- `findstr /i /c:"[B3] Input ownership" server.log client1.log client2.log`
- `findstr /i /c:"[B2] NetworkPlayer spawned" server.log client1.log client2.log`

通过信号（核心）：
- 每个客户端日志中，本地玩家有且仅有一条：`inputEnabled=True` 且 `isOwner=True`
- 同一客户端中，其他玩家应为：`inputEnabled=False` 且 `isOwner=False`
- 实机验证时，仅当前窗口（本地 Owner）输入会驱动自己对应的网络玩家位置变化

失败信号：
- 非 Owner 角色出现 `inputEnabled=True`
- 两个客户端都能同时控制同一个 `owner` 的玩家

### B4 验收日志标准（位姿同步与基础平滑）

建议筛选命令：
- `findstr /i /c:"[B4] Sync config" server.log client1.log client2.log`
- `findstr /i /c:"[B2] NetworkPlayer spawned" server.log client1.log client2.log`

通过信号（核心）：
- 三端都出现 `"[B4] Sync config"`（`interpolate=True`，且 `serverAuth=True`）
- 双客户端移动时，远端玩家位置连续变化，无明显跳点/抽搐（观感验收）
- 双客户端都能持续看到对方位姿更新，不会“卡在出生点”

失败信号：
- 缺失 `"[B4] Sync config"` 日志（说明未挂到 `NetworkTransform`）
- 移动时远端位姿长时间不更新或频繁瞬移
